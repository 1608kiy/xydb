# SOFTWARE + WEB 预想对齐补齐计划

## 1. 目标（按你的原始预想）

- WEB 端：可在浏览器稳定访问与使用。
- SOFTWARE 端：可在官网下载 APK 并在手机安装体验。
- 双端互通：网页端与软件端共享同一后端与数据，账号、任务、番茄、打卡、周报数据一致。
- 可持续发布：后续每次改动都能稳定出包、上线、回滚。

## 2. 当前状态与差距

- 已有：官网可下载 APK，后端与网页端可运行，基础互通可用。
- 差距：当前 APK 主要是 WebView 壳，尚未形成“以 SOFTWARE 目录为核心输入”的标准软件端发布流程。
- 关键缺口：
  - `SFOTWARE/frontend` 改动没有被标准化纳入 APK 构建链路。
  - 缺少“软件端版本号 -> 对应源码提交 -> 对应下载包”的可追溯机制。
  - 缺少一套固定的端到端验收清单，无法快速证明“这次发布完全对齐预想”。

## 3. 总体路线（分三阶段）

- 阶段 A（快速对齐，3-5 天）：把 SOFTWARE 前端作为 APK 的标准输入源，形成可重复打包与发布。
- 阶段 B（质量补齐，5-7 天）：补齐鉴权、异常、版本、监控、回滚能力。
- 阶段 C（体验增强，7-14 天）：补齐移动端体验与必要原生能力，达到“像独立软件端”的使用感。

## 4. 阶段 A：快速对齐（核心）

### 4.1 明确目录与产物映射（1 天）

- 固定映射：
  - 软件端功能源码：`SFOTWARE/frontend/**`
  - 软件端壳工程：`E:\computer science\app\RingNoteApp`
  - 官网下载产物：`/opt/ringnote-app/releases/android/ringnote-debug.apk`
- 输出文档：`版本映射表`（版本号、时间、源码 commit、APK SHA256）。

### 4.2 建立“SOFTWARE -> APK”标准构建链路（1-2 天）

- 构建前步骤：
  - 将 `SFOTWARE/frontend` 同步到线上站点目录（或同步到壳工程本地资源目录，二选一，建议先走线上目录以降低改造风险）。
- 构建步骤：
  - 使用现有安全脚本构建 APK（已具备防高负载能力）。
- 发布步骤：
  - 产物发布到下载目录。
  - 校验下载链接状态码、包大小、SHA256。

### 4.3 发布后最小验收（1 天）

- PC 浏览器验收：登录、任务增删改、日历、番茄钟、打卡、周报、个人中心。
- 手机 APK 验收：同一账号登录后，关键数据与网页端一致。
- 必过标准：
  - 关键链路用例通过率 100%。
  - 数据一致性抽检通过率 100%。

## 5. 阶段 B：质量补齐

### 5.1 版本与可追溯（1 天）

- APK 文件名改为带版本：`ringnote-vX.Y.Z+build.apk`。
- 在官网下载区显示：版本号、发布时间、更新内容、SHA256。
- 每次发布生成 `release-note` 并归档到 `WEB/REPORTS/`。

### 5.2 稳定性与监控（2 天）

- 增加发布后自动健康检查：
  - 官网首页可访问。
  - 关键 API 正常。
  - APK 下载链接可访问且体积非 0。
- 增加故障回滚脚本：
  - 一键回退到上一 APK。
  - 一键回退到上一前端静态资源版本。

### 5.3 安全与配置（1-2 天）

- 清理默认开发安全配置，避免默认密码告警。
- 统一环境配置：开发/预发/生产 API 地址与开关。

## 6. 阶段 C：体验增强（可并行推进）

- WebView 体验优化：启动页、离线提示、弱网重试、错误页兜底。
- 移动端能力增强：
  - 登录态持久化。
  - 通知与本地提醒（番茄钟/打卡）。
  - 必要时加入本地缓存与增量同步。
- 可选路线（中长期）：
  - 从壳应用逐步演进到 Flutter/RN 独立移动端。

## 7. 验收标准（最终“对齐完成”定义）

- 官网可以稳定下载最新 APK。
- APK 安装后可正常使用核心功能。
- 与 WEB 端同账号数据实时互通（任务、番茄、打卡、周报）。
- 每次发布均有版本记录、校验信息、回滚方案。
- 连续 7 天无阻断级问题（P0/P1）。

## 8. 建议排期（从今天起）

- D1-D2：完成阶段 A 的目录映射、构建链路、一次全链路发布。
- D3-D5：完成阶段 A 验收并修复问题。
- D6-D8：完成阶段 B 的版本化、监控、回滚。
- D9-D14：推进阶段 C 的体验增强。

## 9. 今天可立即执行的 5 件事

- 在 `WEB/REPORTS/` 新建发布记录模板（版本、commit、SHA256、验收结果）。
- 固化 `SFOTWARE/frontend` 到线上站点的同步脚本。
- 运行安全构建脚本生成新 APK 并发布。
- 官网下载区补充版本号与更新时间。
- 按“网页 + APK + 数据互通”执行一次全链路回归并出报告。
