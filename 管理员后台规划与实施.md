# 管理员账号与后台管理功能规划（含实施）

## 1. 目标
- 支持固定管理员账号登录：账号 `admin`，密码 `admin`。
- 管理员从登录页登录后，跳转到后台管理页面。
- 管理员可以在后台页面查看用户列表，并删除普通用户。

## 2. 约束与设计原则
- 保持现有 JWT + `/api/me` 的鉴权链路，不引入额外会话体系。
- 最小化数据库变更：不新增角色字段，通过固定管理员邮箱识别管理员身份。
- 接口结果继续使用现有 `Result` 结构（`code/message/data`）。

## 3. 方案设计
### 3.1 管理员身份识别
- 约定管理员内部邮箱：`admin@ringnote.local`。
- 登录接口接收 `email=admin` 或 `email=admin@ringnote.local` 时，走管理员登录分支。
- 若管理员用户不存在，则自动创建管理员账号（昵称：系统管理员）。
- 管理员密码固定为 `admin`（便于当前阶段快速落地）。

### 3.2 后端接口
- 新增接口：`GET /api/admin/users`
  - 作用：获取所有用户信息（脱敏，不返回密码）。
  - 权限：仅管理员。
- 新增接口：`DELETE /api/admin/users/{id}`
  - 作用：删除指定用户。
  - 权限：仅管理员。
  - 保护：管理员账号本身不可删除。

### 3.3 前端交互
- 登录页允许输入 `admin`。
- 登录成功后：
  - 管理员 -> 跳转 `后台管理页面.html`
  - 普通用户 -> 跳转 `待办页面.html`
- 新增后台管理页：
  - 展示用户列表
  - 支持删除普通用户
  - 支持刷新列表、退出登录

## 4. 实施清单（已执行）
- [x] 后端 `AuthService` 增加管理员登录分支与管理员用户自动创建逻辑。
- [x] 后端 `UserService` 增加管理员判断、查询全部用户、按 ID 查询/删除能力。
- [x] 后端 `UserController` 增加 `/api/admin/users`、`/api/admin/users/{id}` 两个接口与权限校验。
- [x] 前端 `登录页面.html` 放开 admin 校验，增加管理员登录后跳转后台逻辑。
- [x] 新增 `后台管理页面.html`，实现用户列表与删除操作。

## 5. 验收步骤
1. 打开登录页，输入账号 `admin`、密码 `admin`。
2. 登录成功后应跳转后台管理页。
3. 后台页加载用户列表，右侧可删除普通用户。
4. 非管理员 token 打开后台页时应被拦截并跳转。

## 6. 风险与后续建议
- 当前管理员密码固定为 `admin`，仅适合内网开发/演示。
- 建议下一步：
  1. 将管理员密码改为环境变量注入；
  2. 引入正式角色字段（如 `role`）并做细粒度 RBAC；
  3. 增加审计日志（谁删除了哪个用户）。
