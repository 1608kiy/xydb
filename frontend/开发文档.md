# 前端开发文档（更新于 2026-03-20）

本文档用于说明当前 `frontend/` 的实现方式、核心交互能力与后续接入规范。

## 1. 工程形态
- 纯静态多页面：HTML + Tailwind CDN + 原生 JavaScript。
- 公共脚本：
  - `common.js`：基础工具、API 请求封装、鉴权检查。
  - `data.js`：AppState 与 localStorage 状态管理。
  - `components.js`：通用组件与导航辅助。
- 页面之间通过链接跳转，不依赖前端构建工具。

## 2. 页面清单
- `登录页面.html`
- `注册页面.html`

- `日历页面.html`
- `番茄钟页面.html`
- `数据周报页面.html`
- `打卡页面.html`

## 3. API 与鉴权约定

### 3.1 请求封装
`common.js` 中 `apiRequest(path, options)` 已支持：
- 自动附带 `Authorization: Bearer <token>`。
- 多候选基址回退：
  1. `window.__API_BASE__`
  2. `localStorage.apiBase`
  3. 同源 origin
  4. `http://localhost:8080`
- 当 `/api/*` 返回 404/405 或非标准 JSON 结构时自动尝试下一个候选基址。

### 3.2 登录态检查
`checkAuthOnLoad()` 用于页面加载时验证 token，失效时跳回登录页。

## 4. 待办页核心能力（已落地）

### 4.1 创建任务降级策略
- 后端正常：直接创建并回填列表。
- 401/403：提示鉴权错误。
- 404/405/5xx 或网络异常：自动离线创建。

### 4.2 离线队列与自动同步
- 本地创建任务写入 `todoPendingCreates_v1` 队列。
- 页面初始化与 `online` 事件触发自动同步。
- 同步成功后用服务端任务替换本地临时任务。

### 4.3 同步状态徽标
- 状态：`待同步` / `同步中` / `已同步`。
- 自动根据队列长度与同步过程更新。

### 4.4 排序功能
- 支持：按时间、按优先级、按标签、按完成状态。
- 排序策略全局生效于今日/明日列表。
- 菜单当前项高亮。

### 4.5 侧边筛选功能
- 主分类支持点击并生效：
  - 所有任务
  - 我的一天
  - 重要任务
  - 已计划
  - 已分配给我
- 标签支持点击筛选，再次点击取消。
- 主分类与标签筛选已做互斥处理，避免叠加造成误解。
- 无结果时显示空状态文案。

## 5. 状态管理说明

### 5.1 localStorage 键
- `ringnote_app_state_v1`：AppState 基础状态
- `todoTasks` / `todoTags`：待办页局部缓存
- `todoPendingCreates_v1`：离线待同步任务队列
- `token`：登录 token
- `apiBase`：可选 API 基址覆盖

### 5.2 任务字段兼容
- 兼容后端 `tags`（字符串 JSON）与前端 `tag`（单值）映射。
- 兼容本地临时任务 ID（如 `local_*`）。

## 6. 调试建议

### 6.1 先确认后端
```powershell
Invoke-WebRequest -Uri "http://localhost:8080/actuator/health" -UseBasicParsing
```

### 6.2 再确认任务接口
```powershell
Invoke-WebRequest -Uri "http://localhost:8080/api/tasks" -UseBasicParsing
```

### 6.3 前端联调重点
- 打开浏览器控制台观察 `apiRequest` 响应结构。
- 检查 `localStorage` 中 `todoPendingCreates_v1` 是否随操作变化。

## 7. 开发约定
- 保持页面间风格一致（顶部、底部导航、提示交互）。
- 新增交互优先复用 `common.js` 工具函数。
- 避免把调试日志或临时目录提交到仓库（如 `backend_run.log`、`tmp/`）。

## 8. 后续完善建议
1. 筛选与排序条件持久化（刷新页面后保持状态）。
2. 将“已分配给我”对齐后端真实字段（assignee）。
3. 离线同步增加冲突策略与重试上限。
4. 多页面逐步组件化，减少重复脚本。
