### 一、技术栈与工程形态

- **技术选型**
  - 纯 HTML + Tailwind CSS（CDN 引入） + 原生 JavaScript。
  - Font Awesome 作为图标库（CDN）。
  - ECharts 用于数据可视化（数据周报、部分日历）。
  - localStorage 用于本地数据持久化。
- **工程结构**
  - 每个业务功能一个独立 HTML 文件，彼此通过 `<a href="*.html">` 跳转。
  - 公共脚本分离：`data.js`（状态管理）、`common.js`（工具函数）、`components.js`（组件渲染）。
  - 无构建工具、无模块打包，适合作为原型或教学/小项目展示。

### 二、公共样式与工具类

- **Tailwind 配置（各页面 `<head>` 的 inline `tailwind.config`）**
  - 统一的颜色：
    - 主色：`primary`、`secondary`、`success`、`warning`、`danger`、`info` 等。
    - 任务标签色：`task-work`、`task-study`、`task-life`、`task-health`。
  - 字体：
    - `fontFamily.inter = ['Inter', 'system-ui', 'sans-serif']`。

- **通用工具类（`<style type="text/tailwindcss">` 定义）**
  - 布局与滚动：
    - `.content-auto`：`content-visibility: auto`。
    - `.scrollbar-hide`：隐藏滚动条。
  - 卡片与悬浮效果：
    - `.task-card-shadow` / `.task-card-shadow-hover`。
    - `.card-shadow` + `.card-hover`：统一与打卡/个人中心风格。
  - 标签与优先级：
    - `.tag-work / .tag-study / .tag-life / .tag-health`。
    - `.priority-high / .priority-medium / .priority-low`。
  - 显隐控制：
    - `.modal-mask.hidden`、`.task-detail-panel.hidden`：通过 `display: none` 实现。

- **字体与整体尺寸**
  - 所有主要功能页 `<body>` 上统一：
    - `class="... text-[13px] md:text-[14px]"`
    - 保证手机端较紧凑，桌面端适中。

### 三、数据管理与状态持久化

#### 3.1 AppState 对象（`data.js`）

- **核心结构**
  ```javascript
  const AppState = {
    user: { id, name, email, phone, level, exp, avatar },
    tasks: [{ id, title, description, dueAt, status, priority, labels, estimate, createdAt, subTasks }],
    pomodoroSessions: [{ startedAt, duration, type }],
    checkins: [{ date, streak }],
    settings: { theme, notifications, pomodoro, calendar },
    labels: [{ id, name, color }]
  };
  ```

- **核心方法**
  - `init()`：从 localStorage 初始化数据
  - `save()`：保存到 localStorage
  - `reset()`：重置为默认数据
  - `logout()`：清除用户状态

- **数据持久化**
  - 使用 `STORAGE_KEY = 'qingyue_todo_app_state_v1'` 作为 localStorage 键
  - JSON 序列化存储，支持热重载时数据恢复

#### 3.2 工具函数（`common.js`）

- **UI 交互**
  - `showToast(message, duration)`：显示提示消息
  - `isMobile()`：检测移动端
  - `safeNavigate(url)`：安全页面跳转

- **日期处理**
  - `formatDateYMD(date)`：格式化为 YYYY-MM-DD
  - `getWeekStart(date)` / `getWeekEnd(date)`：获取周起始/结束日期
  - `isDateInRange(date, start, end)`：日期范围判断
  - `isSameDay(date1, date2)`：同一天判断

- **统计计算**
  - `getWeeklyStats()`：计算周统计数据（完成任务数、专注时长、连续番茄数、效率评分）
  - `getCheckinStats()`：计算打卡统计（连续天数、最长记录、类型分布）

- **任务分组**
  - `nearestTaskGroup(task)`：根据截止时间分组任务（今日/明日/未来7天/以后）

#### 3.3 组件渲染（`components.js`）

- **导航组件**
  - `renderHeader(activePage)`：渲染顶部导航栏
  - `renderFooter(activePage)`：渲染底部 Tab 导航
  - `bindGlobalLogout()`：绑定全局退出登录事件

### 四、导航设计与路由约定

- **登录跳转**
  - `登录页面.html` 登录表单提交：
    - 前端校验账号、密码非空。
    - 直接 `window.location.href = '待办页面.html'`，模拟登录成功后进入待办清单。

- **注册跳转**
  - `注册页面.html` 注册表单提交：
    - 前端校验账号、密码、确认密码一致性。
    - 模拟注册成功，跳转到 `待办页面.html`。

- **底部 Tab 导航（所有功能页一致）**
  - 链接与文件名约定：
    - 待办：`待办页面.html`
    - 日历：`日历页面.html`
    - 番茄钟：`番茄钟页面.html`
    - 数据周报：`数据周报页面.html`
    - 打卡：`打卡页面.html`
    - 个人中心：`个人中心页面.html`
  - 当前页高亮规则：
    - 在对应页面，将自己的 Tab 使用 `text-primary`。
    - 其他 Tab 使用 `text-gray-500 hover:text-primary`。
    - 由于是多页应用，不需要 JS 计算激活态，静态类名即可。

- **顶部返回按钮**
  - 大多数页面的左上角箭头指向 `待办页面.html` 或 `javascript:void(0);`（占位）。
  - 可根据后续需要，调整为返回统一首页。

### 五、各页面实现细节

#### 5.1 登录页（`登录页面.html`）

- **结构**
  - 左侧欢迎区（PC 显示）：品牌名、slogan、示意图与用户数。
  - 右侧表单区：登录表单 + 第三方登录按钮。
  - 底部版权区域。

- **JS 模块**
  - `password-toggle`：密码显示/隐藏切换。
  - `form-submit`：
    - 登录表单：阻止默认提交 -> 非空校验 -> 正则校验 -> 设置用户状态 -> 跳转到待办页。
    - 第三方按钮：弹提示说明选择哪种登录方式。

#### 5.2 注册页（`注册页面.html`）

- **结构**
  - 左侧欢迎区（PC 显示）：同登录页。
  - 右侧表单区：注册表单 + 第三方注册按钮。
  - 底部版权区域。

- **JS 模块**
  - `password-toggle`：密码显示/隐藏切换。
  - `password-strength`：实时检测密码强度，更新进度条和提示文案。
  - `form-submit`：
    - 注册表单：阻止默认提交 -> 非空校验 -> 密码一致性校验 -> 设置用户状态 -> 跳转到待办页。
    - 第三方按钮：弹提示说明选择哪种注册方式。

#### 5.3 待办页（`待办页面.html`）

- **DOM 结构**
  - `<header>`：顶部导航。
  - `aside`（左）：分类与标签列表。
  - `<main>`：快捷操作栏 + 任务分组。
  - `aside`（右）：任务详情编辑侧栏。
  - `<footer>`：底部 Tab 导航。

- **JS 模块**
  - `date-script`：计算当前日期并写入 `#current-date`。
  - `task-card-script`：
    - 监听 `.task-card` 点击，在桌面端显示右侧详情侧栏。
    - 监听 `.task-checkbox` 勾选，切换标题样式（删除线、灰色）并调用 `updateTaskStats`（示例逻辑）。
  - `quick-actions-script`：
    - `#quick-add-task` + `#add-task-btn`：快捷添加任务弹出 alert。
    - 排序/筛选下拉显隐。
    - "添加新标签"按钮弹出 prompt。
  - `share-copy-script`：
    - 分享按钮：alert。
    - 复制按钮：`navigator.clipboard` 复制任务标题。
    - 删除按钮：confirm 后移除对应 `.task-card` 节点。

#### 5.4 日历页（`日历页面.html`）

- **DOM 结构**
  - 顶部导航：今天按钮 + 当前日期 + 视图切换 + 添加任务 + 头像菜单。
  - 主内容：
    - 快捷操作栏。
    - `calendar-container`：
      - `#month-view`
      - `#week-view`
      - `#day-view`
    - 任务池与任务分类说明。
  - 右侧：任务详情侧栏 + 专注历史侧栏。
  - 底部 Tab 导航。

- **JS 模块**
  - `date-script`：同待办。
  - `echarts-script`：
    - 初始化 4 个图表（趋势线 / 分类饼图 / 番茄柱状图 / 热力图）。
    - 注册 `resize` 事件更新尺寸。
  - `calendar-script`：
    - 视图切换：基于 `.view-btn` 的 `data-view` 切换 `.calendar-view` 显隐。
    - 拖拽任务：
      - `.task-draggable` 绑定 `dragstart`，写入任务文本。
      - 日历格子绑定 `dragover/drop`，输出对应任务信息（预留插入任务逻辑）。
    - 日历中任务块的点击事件：控制台输出"打开任务详情"，可扩展为右侧面板联动。

#### 5.5 番茄页（`番茄钟页面.html`）

- **DOM 结构**
  - 顶部导航：返回、标题、副标题、当前模式、日期、"本周专注统计"按钮、头像菜单。
  - 左侧：示例任务列表。
  - 中间：
    - 模式切换按钮（`data-mode="focus|short|long"`）。
    - 圆形计时器（外环 + 内圈 + `#timer-display`）。
    - 任务选择下拉框。
    - 开始/暂停、重置按钮。
    - 今日番茄 / 已完成循环 / 当前番茄序号。
  - 下方：本次会话专注历史 `#session-history`。
  - 右侧：专注小贴士与今日概要。
  - 底部 Tab。

- **JS 模块**
  - `pomodoro-script`：
    - `modeDurations` 定义三种模式秒数。
    - `setMode(mode)` 控制当前模式切换、剩余时间重置、按钮样式更新。
    - `updateTimerUI()`：更新时间文本与进度环角度。
    - `startTimer()/stopTimer()/resetTimer()` 控制 `setInterval` 与 UI。
    - `completePomodoro()`：
      - 仅在专注模式完成时更新统计计数。
      - 每完成 1 个专注番茄增加"今日番茄数"和"当前番茄序号"；每 4 个番茄增加 1 个循环。
      - 将完成记录追加到 `#session-history`。
    - 绑定事件：
      - 模式按钮点击。
      - 开始/暂停按钮点击。
      - 重置按钮点击。
      - 任务选择变更时更新"当前任务"显示。
      - 清空历史按钮清空列表。

#### 5.6 数据周报页（`数据周报页面.html`）

- **DOM 结构**
  - 顶部导航：标题、日期 + 范围选择器、导出、分享按钮、头像菜单。
  - 主体：
    - 卡片概览区。
    - 趋势图 + 分类饼图 + 番茄柱状图 + 热力图。
    - 效率时段分析卡片。
    - 智能建议列表。
  - 右侧：任务详情侧栏 + 专注历史侧栏。
  - 底部 Tab。

- **JS 模块**
  - `date-script`：日期显示。
  - `echarts-script`：初始化所有图表、绑定 resize。
  - 任务卡片交互 / 快捷操作 / 番茄脚本 / 分享复制脚本在本页也有复用，保持交互体验一致。

#### 5.7 打卡页（`打卡页面.html`）

- **实现方式**
  - 顶部导航同其他页面风格。
  - 今日打卡区 + 文本备注 + 按钮，更新状态文案。
  - 打卡日历用 JS 根据当月日期构建网格，今天和过去日期高亮。
  - 最近打卡记录在当前会话内使用 DOM 增删（不持久化）。
  - 统计概要与类型分布为静态示例，方便后续替换真实数据。

#### 5.8 个人中心页（`个人中心页面.html`）

- **实现方式**
  - 左侧：用户信息卡片 + 数据概要 + 存储管理 + 帮助与关于。
  - 右侧：账号安全 + 应用设置 + 日历 & 数据隐私 + 退出/注销 + 徽章墙。
  - 暂无真实数据源，均为示例文案与 UI 结构，便于后端接入时替换成真实数据。

### 六、后续演进建议

1. **组件化与模板化**
   - 将顶部导航、底部 Tab、用户头像菜单、卡片组件抽取为可复用片段，避免在多个 HTML 中重复维护。

2. **数据抽象与状态管理**
   - 把任务、番茄记录、打卡记录等统一抽象成 JS 对象或 JSON 数据，所有 UI 改由渲染函数生成。
   - 在未接后端前，可使用 `localStorage` 存储核心数据。

3. **前端路由与 SPA 重构**
   - 如需更好的体验，可引入 Vue/React + Router，将六个 Tab 页改造成单页应用的六个路由视图。

4. **与后端集成**
   - 登录 / 注册 / 任务 CRUD / 统计数据 / 打卡记录等改为通过 API 获取与提交。
   - 在当前静态页面结构基础上，尽量保留 DOM 结构与类名，降低重构成本。

5. **性能优化**
   - 引入虚拟滚动处理大量任务列表。
   - 图表按需加载，减少初始加载时间。
   - 实现数据缓存和增量更新机制。

6. **用户体验增强**
   - 添加离线支持（Service Worker）。
   - 实现数据同步和冲突解决。
   - 增加键盘快捷键支持。
   - 优化移动端触摸交互。
