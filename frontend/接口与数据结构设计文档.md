### 二、认证与用户接口

#### 1. 登录

- **POST** `/api/auth/login`

```json
{
  "account": "user@example.com",
  "password": "******"
}
```

**响应**

```json
{
  "token": "jwt_or_session_id",
  "user": { /* User 对象 */ }
}
```

#### 2. 注册

- **POST** `/api/auth/register`

```json
{
  "nickname": "张明",
  "account": "user@example.com",
  "password": "******"
}
```

**响应**

```json
{
  "user": { /* User */ },
  "token": "jwt"
}
```

#### 3. 获取当前用户

- **GET** `/api/me`

返回当前登录用户信息及其设置。

### 四、任务与标签接口

#### 1. 获取任务列表

- **GET** `/api/tasks`

支持 query：

- `status=pending|completed`
- `list=all|myday|planned|important|assigned`
- `tag=工作`
- `date=2026-03-13`

**响应**

```json
[
  { /* Task */ },
  { /* Task */ }
]
```

#### 2. 创建任务

- **POST** `/api/tasks`

```json
{
  "title": "完成项目提案PPT",
  "description": "",
  "priority": "high",
  "tags": ["工作"],
  "dueAt": "2026-03-13T14:00:00Z",
  "list": "所有任务"
}
```

#### 3. 更新任务

- **PUT** `/api/tasks/{taskId}`  
  允许部分字段更新（标题、描述、状态、时间、列表、标签等）。

#### 4. 删除任务

- **DELETE** `/api/tasks/{taskId}`

#### 5. 子任务更新

- **PUT** `/api/tasks/{taskId}/subtasks`

```json
{
  "subTasks": [
    { "id": "st_1", "title": "收集市场数据", "completed": true },
    { "id": "st_2", "title": "分析竞争对手", "completed": false }
  ]
}
```

#### 6. 标签管理

- **GET** `/api/tags` 获取标签列表  
- **POST** `/api/tags` 创建自定义标签

```json
{
  "name": "考研",
  "color": "#8B5CF6"
}
```

### 五、番茄钟与专注记录接口

#### 1. 创建番茄记录

- **POST** `/api/pomodoros`

```json
{
  "taskId": "t_001",
  "mode": "focus",
  "plannedMinutes": 25,
  "actualMinutes": 25,
  "startedAt": "2026-03-13T09:00:00Z",
  "endedAt": "2026-03-13T09:25:00Z",
  "completed": true
}
```

#### 2. 查询番茄记录

- **GET** `/api/pomodoros?start=2026-03-10&end=2026-03-16`

返回 `PomodoroSession` 列表，用于数据周报与番茄统计。

### 八、前后端分工建议

- **前端**
  - 使用当前 HTML + JS 结构，通过封装 API 调用上述接口。
  - 在未接入后端前，可用 `localStorage` 或内存 Mock 实现同样的数据结构。

- **后端**
  - 负责认证、数据持久化和聚合统计逻辑。
  - 对外暴露与上文一致的 REST 接口，方便前端无缝切换数据源。
