# XYDB 操作指南

## 1. 日常高频命令

### 1.1 查看后端状态
```bash
systemctl status xydb-backend --no-pager
systemctl is-active xydb-backend
```

### 1.2 重启后端
```bash
systemctl restart xydb-backend
```

### 1.3 查看后端实时日志
```bash
journalctl -u xydb-backend -f
```

### 1.4 查看最近日志（排障）
```bash
journalctl -u xydb-backend -n 200 --no-pager
```

### 1.5 检查 Nginx 配置并重载
```bash
nginx -t
systemctl reload nginx
```

## 2. 日常巡检流程（建议每天）

1. 检查后端进程状态：
```bash
systemctl is-active xydb-backend
```

2. 检查服务监听：
```bash
ss -lntp | grep 8080
```

3. 检查后端健康接口：
```bash
wget -S -O /dev/null http://127.0.0.1:8080/actuator/health 2>&1 | head -n 12
```

4. 检查网页入口：
```bash
wget -S -O /dev/null http://127.0.0.1/ 2>&1 | head -n 12
```

5. 检查数据库可访问：
```bash
mysql -uxydb -p -D xydb -e 'SHOW TABLES;'
```

## 3. 发布新版本操作

### 3.1 发布前
1. 备份 Nginx 配置：
```bash
ts=$(date +%Y%m%d_%H%M%S)
cp /etc/nginx/sites-available/xydb /etc/nginx/sites-available/xydb.bak.$ts
```

2. 备份数据库：
```bash
mysqldump -uxydb -p xydb > /opt/xydb-app/backup_xydb_$(date +%Y%m%d_%H%M%S).sql
```

### 3.2 发布中
1. 上传新代码到 /opt/xydb-app/current
2. 打包后端：
```bash
cd /opt/xydb-app/current/backend
./mvnw -q -DskipTests package
```

3. 重启后端：
```bash
systemctl restart xydb-backend
```

4. 如有 Nginx 改动，执行：
```bash
nginx -t
systemctl reload nginx
```

### 3.3 发布后
1. 等待 20~40 秒后执行健康检查
2. 访问首页与登录
3. 验证关键接口：登录、/api/me、任务创建

## 4. 快速故障处理

### 4.1 网页打开正常但接口失败
1. 先看后端状态：
```bash
systemctl status xydb-backend --no-pager
```
2. 看后端日志：
```bash
journalctl -u xydb-backend -n 200 --no-pager
```
3. 看数据库连接是否异常（日志里检索 Hikari、MySQL、Exception）

### 4.2 出现 502 Bad Gateway
1. 检查 8080 是否监听：
```bash
ss -lntp | grep 8080
```
2. 如未监听，重启后端并复查日志
3. 确认 Nginx 反代地址是 127.0.0.1:8080

### 4.3 重启后端后短时间失败
- 后端首次初始化可能较慢，等待 20~40 秒后再做探活

## 5. 回滚操作

### 5.1 回滚 Nginx
1. 查看备份：
```bash
ls -la /etc/nginx/sites-available/xydb.bak*
```
2. 恢复备份并重载：
```bash
cp /etc/nginx/sites-available/xydb.bak.时间戳 /etc/nginx/sites-available/xydb
nginx -t
systemctl reload nginx
```

### 5.2 回滚后端
1. 切换到上一版 jar（需提前保留）
2. 重启服务：
```bash
systemctl daemon-reload
systemctl restart xydb-backend
```

### 5.3 回滚数据库
```bash
mysql -uxydb -p xydb < /opt/xydb-app/某次备份.sql
```

## 6. 账号与配置管理建议
- 生产环境密码、密钥统一放在 /etc/xydb/backend.env
- 修改配置后务必重启后端服务
- 不要把生产密码提交到仓库

## 7. 一键检查清单
每次变更后至少确认以下 5 项：
1. systemctl is-active xydb-backend = active
2. nginx -t = syntax is ok
3. /actuator/health 返回 200
4. 首页可访问
5. 登录与创建任务接口可用
