# XYDB 完整部署文档

## 1. 文档目标
本文档用于指导将 XYDB 项目完整部署到 Ubuntu 服务器，包含：
- 后端 Spring Boot 服务部署
- 前端静态页面部署
- MySQL 数据库接入
- Nginx 反向代理与 80 端口发布
- 服务启动、自启动、验证与回滚

## 2. 本次部署结果（已验证）
- 服务器公网地址：47.108.170.112
- 前端入口：http://47.108.170.112/
- 后端对内端口：8080
- 后端服务名：xydb-backend（systemd）
- 数据库：MySQL 8，本机实例，业务库为 xydb

## 3. 部署目录与关键文件
- 项目目录：/opt/xydb-app/current
- 后端目录：/opt/xydb-app/current/backend
- 前端目录：/opt/xydb-app/current/frontend
- 后端环境文件：/etc/xydb/backend.env
- 后端服务文件：/etc/systemd/system/xydb-backend.service
- Nginx（80）站点：/etc/nginx/sites-available/xydb
- Nginx（备用）站点：/etc/nginx/conf.d/xydb_7833.conf

## 4. 服务器环境要求
- 操作系统：Ubuntu 24.04+
- 已开放端口：22、80、7833
- 权限要求：root 或具备 sudo 权限

## 5. 首次部署步骤

### 5.1 上传项目代码
从本地上传 backend 和 frontend 到服务器：

```bash
mkdir -p /opt/xydb-app/current
# 上传 backend/ 与 frontend/ 到 /opt/xydb-app/current
```

上传后目录应为：
- /opt/xydb-app/current/backend
- /opt/xydb-app/current/frontend

### 5.2 安装 Java 21

```bash
apt-get update -y
apt-get install -y openjdk-21-jdk
java -version
```

### 5.3 安装并启动 MySQL 8

```bash
apt-get update -y
apt-get install -y mysql-server
systemctl enable mysql
systemctl restart mysql
systemctl is-active mysql
```

### 5.4 初始化业务数据库与账号

```sql
CREATE DATABASE IF NOT EXISTS xydb DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_0900_ai_ci;
CREATE USER IF NOT EXISTS 'xydb'@'localhost' IDENTIFIED BY '请替换为强密码';
CREATE USER IF NOT EXISTS 'xydb'@'127.0.0.1' IDENTIFIED BY '请替换为强密码';
ALTER USER 'xydb'@'localhost' IDENTIFIED BY '请替换为强密码';
ALTER USER 'xydb'@'127.0.0.1' IDENTIFIED BY '请替换为强密码';
GRANT ALL PRIVILEGES ON xydb.* TO 'xydb'@'localhost';
GRANT ALL PRIVILEGES ON xydb.* TO 'xydb'@'127.0.0.1';
FLUSH PRIVILEGES;
```

可通过以下命令执行：

```bash
mysql -uroot
```

### 5.5 打包后端

```bash
cd /opt/xydb-app/current/backend
chmod +x mvnw
./mvnw -q -DskipTests package
ls -lah target/*.jar
```

预期产物：
- target/backend-0.0.1-SNAPSHOT.jar

### 5.6 配置后端环境变量
创建文件 /etc/xydb/backend.env：

```env
SPRING_DATASOURCE_URL=jdbc:mysql://127.0.0.1:3306/xydb?serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
SPRING_DATASOURCE_USERNAME=xydb
SPRING_DATASOURCE_PASSWORD=请替换为数据库密码
JWT_SECRET=请替换为至少32字节强密钥
```

### 5.7 配置 systemd 后端服务
创建文件 /etc/systemd/system/xydb-backend.service：

```ini
[Unit]
Description=XYDB Backend Service
After=network.target mysql.service

[Service]
Type=simple
WorkingDirectory=/opt/xydb-app/current/backend
EnvironmentFile=/etc/xydb/backend.env
ExecStart=/usr/bin/java -jar /opt/xydb-app/current/backend/target/backend-0.0.1-SNAPSHOT.jar --server.port=8080
SuccessExitStatus=143
Restart=always
RestartSec=5
User=root

[Install]
WantedBy=multi-user.target
```

启用并启动：

```bash
systemctl daemon-reload
systemctl enable xydb-backend
systemctl restart xydb-backend
systemctl status xydb-backend --no-pager
```

### 5.8 配置 Nginx 反代（80 端口）
编辑 /etc/nginx/sites-available/xydb：

```nginx
server {
    listen 80;
    server_name 47.108.170.112;

    root /opt/xydb-app/current/frontend;
    index 登录页面.html;
    charset utf-8;

    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location / {
        try_files $uri $uri/ /登录页面.html;
    }
}
```

生效配置：

```bash
nginx -t
systemctl reload nginx
```

## 7. 升级发布流程（后续版本）

1. 上传新代码到 /opt/xydb-app/current
2. 重新打包后端
3. 重启后端服务
4. 验证健康检查
5. 验证前端入口与关键 API

示例命令：

```bash
cd /opt/xydb-app/current/backend
./mvnw -q -DskipTests package
systemctl restart xydb-backend
sleep 8
systemctl is-active xydb-backend
wget -S -O /dev/null http://127.0.0.1:8080/actuator/health 2>&1 | head -n 12
```

## 8. 回滚方案

### 8.1 Nginx 配置回滚
每次改配置前先备份：

```bash
ts=$(date +%Y%m%d_%H%M%S)
cp /etc/nginx/sites-available/xydb /etc/nginx/sites-available/xydb.bak.$ts
```

回滚：

```bash
cp /etc/nginx/sites-available/xydb.bak.具体时间戳 /etc/nginx/sites-available/xydb
nginx -t
systemctl reload nginx
```

### 8.2 后端版本回滚
建议每次发布前保留旧 jar，回滚时改回旧 jar 启动：

```bash
# 例如保留 old.jar
cp /opt/xydb-app/current/backend/target/backend-0.0.1-SNAPSHOT.jar /opt/xydb-app/current/backend/target/backend-old.jar
```

修改服务文件 ExecStart 指向旧 jar 后：

```bash
systemctl daemon-reload
systemctl restart xydb-backend
```

### 8.3 数据库回滚
上线前先做备份：

```bash
mysqldump -uxydb -p xydb > /opt/xydb-app/backup_xydb_$(date +%Y%m%d_%H%M%S).sql
```

恢复：

```bash
mysql -uxydb -p xydb < /opt/xydb-app/某个备份.sql
```

## 9. 常见问题处理

### 9.1 80 端口返回 502
- 检查后端是否启动：systemctl status xydb-backend
- 检查 8080 是否监听：ss -lntp | grep 8080
- 查看后端日志：journalctl -u xydb-backend -n 120 --no-pager

### 9.2 登录后接口 401/403
- 确认请求头是否带 Authorization: Bearer token
- 检查 token 是否过期
- 检查 JWT_SECRET 是否变更导致旧 token 失效

### 9.3 后端启动慢导致短暂不可用
- 属于首次连接数据库和 JPA 初始化的常见现象
- 建议重启后等待 20~40 秒再做健康探测

## 10. 安全建议
- 及时替换默认密码与密钥
- 禁止在文档或代码中明文存储生产密码
- 定期备份数据库并验证恢复流程
- 控制 SSH 源 IP 与登录权限
