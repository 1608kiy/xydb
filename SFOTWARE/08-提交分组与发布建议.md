## 3. 可直接执行的命令（PowerShell）

> 说明：以下命令按分组提交。若你希望先检查，可在每组 `git add` 后先执行一次 `git status -sb`。

```powershell
git add -- "SFOTWARE/frontend/js/pages/登录页面.inline-03.js" "SFOTWARE/frontend/js/pages/注册页面.inline-03.js"
git commit -m "feat(auth): remove local fallback and enforce backend-first login/register"

git add -- "SFOTWARE/frontend/js/pages/待办页面.inline-02.js"
git commit -m "feat(tasks): sync tags and task updates with backend APIs"

git add -- "SFOTWARE/frontend/js/pages/日历页面.calendar-script.js"
git commit -m "feat(calendar): enable server sync for task CRUD subtasks and ai scheduling"

git add -- "SFOTWARE/frontend/js/pages/番茄钟页面.inline-03.js" "SFOTWARE/frontend/js/pages/打卡页面.inline-05.js"
git commit -m "feat(focus-checkin): persist only after backend confirmation"

git add -- "SFOTWARE/frontend/js/pages/数据周报页面.report-data-fetch.js" "SFOTWARE/frontend/js/pages/数据周报页面.inline-06.js" "SFOTWARE/frontend/js/pages/个人中心页面.inline-04.js"
git commit -m "feat(report-profile): fetch remote metrics and harden profile update flow"

git add -- "SFOTWARE/04-接口联调与数据互通方案.md" "SFOTWARE/07-联调验收与问题清单.md" "SFOTWARE/08-提交分组与发布建议.md" "WEB/scripts/run_min_acceptance.ps1" "SFOTWARE/README.md"
git commit -m "docs(test): add acceptance evidence and delivery checklist"

git add -- ".gitignore"
git commit -m "chore(git): ignore e2e dependency artifacts under WEB/scripts"
```

## 4. 注意项

- `.vscode/settings.json` 是否提交，请按团队规范决定；若不提交，可加入 `.gitignore` 或仅保留本地。
- 提交前建议再执行一次：
  - `git status -sb`
  - 关键脚本回归：`WEB/scripts/run_min_acceptance.ps1`
