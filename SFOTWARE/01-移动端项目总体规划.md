# 01 移动端项目总体规划

## 1. 项目目标

### 1.1 业务目标

- 基于当前铃记 WEB 项目，开发“功能一摸一样”的手机 App（iOS/Android）。
- 手机端与网页端数据完全互通，用户可跨端无缝使用。
- 首版上线以“功能一致性 + 稳定性”优先，不追求首版过度创新。

### 1.2 交付目标

- 交付一套可发布的移动端代码仓（推荐独立仓库，便于 CI/CD）。
- 交付接口联调文档、测试用例、上线回滚预案。
- 交付版本节奏：MVP 首发 + 两个稳定迭代版本。

## 2. 现状抽象（来自当前 WEB 项目）

当前核心能力已具备：

- 账号体系：注册、登录、找回密码、获取当前用户。
- 任务体系：任务 CRUD、子任务更新、标签、状态与优先级。
- 专注体系：番茄记录创建与查询。
- 打卡体系：打卡创建、日历查询、最近记录。
- 报表体系：overview、daily-trend、task-category。
- 管理体系：管理员用户列表、删除、批量删除、创建管理员。

关键后端接口分组：

- `/api/auth/*`
- `/api/me`
- `/api/tasks*`
- `/api/tags*`
- `/api/pomodoros*`
- `/api/checkins*`
- `/api/reports*`
- `/api/admin/users*`
- `/api/ai/chat`

## 3. 移动端产品边界

### 3.1 首发必须包含（与 WEB 对齐）

- 登录页、注册页
- 待办清单页
- 日历视图页
- 番茄钟页
- 数据周报页
- 打卡页
- 个人中心页

### 3.2 二期可增强（不阻断首发）

- 系统通知（任务提醒、番茄结束提醒）
- 后台推送（Push）
- 更完整的离线缓存与冲突处理
- 深色模式细粒度定制

## 4. 跨端互通原则

- 单一数据源：全部业务数据以后端数据库为准。
- 单一鉴权：移动端与网页端共用 JWT 规则与过期策略。
- 单一业务口径：任务状态、优先级、统计口径一致。
- 单一时区口径：统一使用 Asia/Shanghai，并保留 UTC 存储能力。

## 5. 组织与角色建议

- 产品负责人：负责“WEB 与移动一致性”验收。
- 移动端开发：2 人（页面 + 状态管理 + API 对接）。
- 后端开发：1 人（接口补齐、性能与安全加固）。
- 测试：1 人（接口测试 + 真机回归）。
- 运维：1 人（发布流程、监控告警、回滚）。

## 6. 关键成功标准

- 同一账号在网页端新增任务，移动端 3 秒内可见。
- 移动端番茄记录在网页周报中正确统计。
- 跨端登录态稳定，24 小时内无异常退出率 > 1%。
- 首发核心流程（登录->建任务->番茄->周报）通过率 99%+。
