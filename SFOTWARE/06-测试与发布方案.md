# 06 测试与发布方案

## 1. 测试策略

### 1.1 测试分层

- 单元测试：状态管理、工具函数、数据转换。
- 集成测试：页面 + API + 本地缓存协作。
- 端到端测试：登录、建任务、番茄、打卡、看周报全链路。

### 1.2 关键回归场景

- 登录态过期后的自动跳转与重新登录。
- 任务创建/更新/删除跨端一致性。
- 番茄记录与报表统计一致性。
- 弱网下请求失败重试与用户提示。

## 2. 发布流程

- 环境顺序：dev -> staging -> prod。
- 发布前检查：
  - P0 功能回归通过。
  - 崩溃率基线可接受。
  - 关键接口 SLA 正常。
- 发布方式：
  - Android 先灰度（20% -> 50% -> 100%）。
  - iOS 通过 TestFlight 分批。

## 3. 监控与告警

- 监控指标：
  - 崩溃率。
  - 接口错误率。
  - 首页加载耗时。
  - 登录成功率。
- 告警阈值：
  - 任一核心接口 5xx > 2% 持续 5 分钟触发告警。
  - 登录成功率 < 98% 触发告警。

## 4. 回滚预案

- 客户端回滚：暂停灰度、回退上一稳定版本。
- 服务端回滚：回退到上一可用后端包。
- 数据回滚：仅在必要时做增量补偿，避免直接删库回滚。

## 5. 验收标准（上线门槛）

- P0 功能全部通过。
- 与 WEB 端核心数据互通验证通过。
- 无阻断级缺陷（P0/P1）。
- 线上健康监控项全部可观测。
