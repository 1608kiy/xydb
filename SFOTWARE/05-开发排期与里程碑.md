# 05 开发排期与里程碑（建议 10 周）

## 1. 总体节奏

- 第 1-2 周：项目搭建与认证模块。
- 第 3-4 周：任务与标签模块。
- 第 5-6 周：日历与番茄模块。
- 第 7 周：打卡与周报模块。
- 第 8 周：个人中心与设置。
- 第 9 周：联调回归与性能优化。
- 第 10 周：灰度发布与正式上线。

## 2. 里程碑定义

### M1（第 2 周末）

- 完成 App 基础框架、登录注册、会话管理。
- 能打通 `auth` + `me` 接口。

### M2（第 4 周末）

- 完成任务列表与增删改查、标签、子任务。
- 跨端任务互通验收通过。

### M3（第 6 周末）

- 完成日历视图与番茄主流程。
- 番茄数据可进入报表链路。

### M4（第 8 周末）

- 完成打卡、周报、个人中心核心功能。
- 首发功能闭环可演示。

### M5（第 10 周末）

- 完成稳定性与性能优化。
- 完成灰度、发布、监控与回滚预案演练。

## 3. 任务拆分建议（按角色）

### 移动端

- UI 还原与交互。
- 状态管理与本地缓存。
- API 接入与错误处理。

### 后端

- 接口补齐与字段一致性。
- 认证与权限校验加固。
- 统计口径稳定。

### 测试

- 接口自动化。
- 真机冒烟与回归。
- 发布验收。

## 4. 风险与缓冲

- 风险 1：WEB 与后端存在局部字段不一致。
- 风险 2：移动端与 WEB 的时间格式处理差异。
- 风险 3：管理员与普通用户权限边界不清晰。

缓冲建议：

- 每两周保留 1-2 天风险处理缓冲。
- 对 P0 功能先做可用，再逐步优化体验。
